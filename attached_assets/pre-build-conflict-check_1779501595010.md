The site currently has 28 hardcoded starter templates.
The new Template Browser (Spec 1) replaces them entirely.

After you locate them in your conflict report above,
do the following as part of Spec 1 — not before:

  → Remove the 28 templates from the DOM in app.html.
    If they are in a container element, remove the
    entire container. If they are generated by a script,
    comment out or remove that generation logic.
  → If there is a button or entry point that opened or
    displayed the 28 templates, repurpose it:
    change its ID to pmg-template-browser-btn,
    change its label to "Templates", and leave it in
    place. Do not add a second button.
  → If the 28 templates had their own CSS, remove or
    comment it out entirely.
  → Do not delete any other feature. Only the 28
    hardcoded template entries and their container.

The site may already have a compare or "see what's
different" feature for prompts. Before building Spec 2:

  → Find it in the conflict report.
  → If the existing feature compares two versions of
    the SAME prompt (before/after edits): it does NOT
    conflict with Spec 2. Spec 2 compares the same
    prompt across THREE MODEL STYLES. They are different

    STOP. Do not build Spec 2. Report the conflict

    #result-top-actions: report its exact ID and

    replacing it.

For every spec in this session and all future sessions:

If you discover ANY of the following while building,
STOP immediately and report before continuing:

  - An existing feature that does the same thing as
    what you are about to build.
  - An existing ID, class, or function name that
    conflicts with a new one in the spec.
  - An existing route in ai.ts that matches a new
    route you are about to add.
  - Any change that would require modifying pmg-ux.js,
    pmg-chassis-v3.js, or any existing script not
    listed under MODIFY in the spec.
  - Any change that would affect billing, auth,
    paywall, or the Stripe/Supabase integration.
  - Anything that is unclear in the spec and could
    go two different ways.

Do not make a judgment call on conflicts.
Do not build around them silently.
Always stop and report. Wait for instruction.
