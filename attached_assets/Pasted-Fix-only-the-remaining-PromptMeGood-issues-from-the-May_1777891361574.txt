Issue 5: Hidden Focusable Controls
Problem:
Audit found hundreds of zero-size hidden controls that are still focusable. This affects keyboard navigation, screen readers, and automated testing.

Fix:
- Audit hidden/inactive UI including:
  - hidden tour controls
  - hidden image wizard controls
  - hidden prompt controls
  - hidden search/menu controls
  - hidden modal buttons
  - hidden duplicate textareas
- Any element that is hidden, zero-size, inactive, or offscreen because its panel/modal is closed should not be keyboard-focusable.
- Use the appropriate approach:
  - remove from DOM when not needed,
  - `hidden`,
  - `display: none`,
  - `inert`,
  - `aria-hidden="true"` plus `tabindex="-1"` for descendants,
  - `disabled` for inactive buttons/inputs.
- Do not use only opacity or transform to hide interactive controls if they remain focusable.
- Ensure closed panels/modals/wizards are not in tab order.

Acceptance:
- Keyboard tab order only reaches visible active controls.
- Hidden zero-size buttons, links, inputs, and textareas are not focusable.
- Screen readers should not announce closed/inactive controls.
- Visible controls still work.
