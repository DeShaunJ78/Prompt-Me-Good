PRODUCT PRINCIPLE:
Beginner Mode = fast first win.
Expert Mode = senior prompt engineer playground.

Expert Mode should be powerful, but hidden until requested. It should feel like a “tweak me” haven for advanced users, not a junk drawer of buttons.

- Main prompt box stays primary.

INTELLIGENT HYBRID BEHAVIOR:
Expert Mode should not be a 100% manual form or giant settings panel.

It should behave like an intelligent prompt-engineering assistant using:
- User input
- Guided options
- Smart recommendations
- One-click expert fixes
- Optional custom fields

The goal is for Expert Mode to feel like:
“I reviewed your prompt. Here is what’s missing. Here are the smartest upgrades. Pick what you want.”

Do not force users to manually fill every section.

Use three levels of intelligence:
1. Guided choices:
   Chips, toggles, sliders, presets, and Off / Auto / Custom states.

2. Smart recommendations:
   Expert Mode recommends missing context, better structure, output format, tone, constraints, or next steps based on the current prompt.

3. One-click expert rewrites:
   Buttons like “Fix This Like A Prompt Engineer” apply professional prompt-engineering upgrades automatically.

UX CONTAINER:
On mobile:
- Expert Command Center opens as a full-screen drawer/overlay.
- It should not shift the page.
- It should have a clear X close button.
- It should trap focus while open.
- It should be scrollable inside the drawer.
- It must not create horizontal overflow.

On desktop:
- Open as a right-side drawer or large overlay.
- Keep the main page clean behind it.
- Escape should close it.
- Closing should return focus to the existing Expert Mode button.

Do not create a separate full page unless absolutely necessary.

TABS:
Use 5 tabs inside Expert Command Center:

1. Diagnose
2. Engineer
3. Tune
4. Variations
5. Save

TAB 1: DIAGNOSE
Purpose:
Expert Mode analyzes the current prompt and tells the user what is weak, missing, risky, or vague.

This tab should be mostly intelligent, not manual.

Show:
- Prompt Strength Score
- Biggest Weakness
- Missing Context
- Risk Warnings
- Recommended Upgrade
- One-click fixes

Score categories:
- Clarity
- Context
- Specificity
- Output Format
- Constraints
- Audience Fit
- Actionability

Example:
Prompt Strength Score:
72/100

Biggest Weakness:
Your prompt asks for marketing help but does not define the audience, offer, platform, or desired action.

Recommended Upgrade:
Add audience, platform, offer, and CTA structure.

Buttons:
- Fix This Like A Prompt Engineer
- Ask Me 5 Questions
- Auto-Fill Safe Defaults
- Add Missing Context
- Pressure Test Prompt

Risk warnings to detect:
- Too vague
- Too broad
- Too many goals
- No audience
- No output format
- Weak constraints
- Conflicting instructions
- Likely generic answer
- Needs source material
- Should be split into a prompt chain

Anti-vague language detection should live inside Diagnose.
Detect phrases like:
- maybe
- kind of
- something about
- help me with
- make it better
- good
- nice
- interesting
- professional
- engaging

Do not make Anti-Vague Language Cleaner a separate tab.

TAB 2: ENGINEER
Purpose:
Let advanced users control the prompt DNA.

This tab should be hybrid:
- Guided controls for most users.
- Optional custom fields for power users.

Use Off / Auto / Custom states where possible.

Prompt DNA sections:
- Role
- Objective
- Audience
- Context
- Constraints
- Tone
- Output Format
- Examples
- Avoid Rules
- Next Steps
- Success Criteria
- Clarifying Questions First

Behavior:
- Auto means PromptMeGood recommends and fills the section intelligently.
- Custom lets the user type or edit.
- Off removes the section from the final prompt.

Include:
- Prompt DNA Summary
- Live optimized prompt preview
- Apply This Structure button
- Reset To Simple Prompt button

Buttons:
- Apply This Structure
- Add Success Criteria
- Add Avoid Rules
- Ask Clarifying Questions First
- Reset To Simple Prompt

TAB 3: TUNE
Purpose:
Let users control how the AI behaves, writes, thinks, and responds.

This tab should be guided, using toggles, chips, sliders, and presets.

Behavior controls:
- Ask Questions First
- Assume Reasonable Defaults
- Challenge My Assumptions
- Give Direct Answer First
- Show Reasoning Summary
- Use Examples
- Avoid Generic Advice
- Prioritize Execution
- Make It More Human
- Make It Less Robotic

Voice/depth controls:
- Direct ↔ Warm
- Simple ↔ Detailed
- Casual ↔ Professional
- Safe ↔ Bold
- Plain ↔ Creative
- Short ↔ Deep

Each control must translate into actual prompt instructions behind the scenes.

Examples:
Bold:
Use direct language. Avoid hedging. Lead with the strongest point.

Simple:
Explain plainly. Avoid jargon. Use short sentences.

Detailed:
Add useful context, examples, and practical next steps.

Prioritize Execution:
Focus on actions, decisions, and next steps instead of theory.

Do not build meaningless sliders. Every control must affect the generated prompt instructions.

TAB 4: VARIATIONS
