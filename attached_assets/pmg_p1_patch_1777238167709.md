## Fix 3 — Usage counter: never show "0+"

**Find this HTML** in the hero section:

```html
<p id="usageCounter" style="display:none; ...">
  <strong id="usageCount">0</strong>+ prompts generated
</p>
```

**It's already `display:none` by default — good.** The bug is in the JS that shows it. Find your `loadUsageCounter` function:

```js
async function loadUsageCounter() {
  const res = await fetch('/api/stats');
  const data = await res.json();
  const total = (data.promptCount || 0) + (data.runCount || 0);

  if (total > 100) {  // ← this threshold is right
    const el = document.getElementById('usageCounter');
    const count = document.getElementById('usageCount');
    if (el && count) {
      count.textContent = total.toLocaleString();
      el.style.display = 'block';  // ← only reveals when real data exists
    }
  }
  // If total <= 100: element stays hidden. Good.
}
```

**The logic is correct — but verify one thing:** make sure there's no other place in the HTML or JS that sets `usageCount` to "0" or makes `usageCounter` visible before this function runs. Search the file for `usageCounter` and `usageCount` — if any other code touches those elements, add a guard.

**Also add error handling** so a failed fetch never accidentally reveals the counter:

```js
async function loadUsageCounter() {
  try {
    const res = await fetch('/api/stats');
    if (!res.ok) return; // ← bail silently on any error
    const data = await res.json();
    const total = (data.promptCount || 0) + (data.runCount || 0);
    if (total > 100) {
      document.getElementById('usageCount').textContent = total.toLocaleString();
      document.getElementById('usageCounter').style.display = 'block';
    }
  } catch (_) {
    // Fail silently — never show "0+"
  }
}
```

**Time to ship:** 5 minutes.

## Fix 4 — "This week's focus": eliminate the loading state

**Find your existing weekly focus code.** It likely does a fetch or async call. Replace the entire function with this static rotation — zero network dependency:

```js
function loadWeeklyFocus() {
  const focuses = [
    "Write a cold email that actually gets a response",
    "Create a 30-day content plan for my Instagram",
    "Build a simple landing page for my freelance service",
    "Find three realistic ways to make $500 extra this month",
    "Write a job description that attracts the right candidates",
    "Plan a product launch for a digital course",
    "Turn my expertise into a consulting pitch",
    "Create a week of TikTok hooks for my niche",
    "Write a sales page for my coaching program",
    "Identify the fastest path to my first 100 customers",
    "Draft a partnership proposal email",
    "Build a prompt that writes viral Twitter threads",
  ];

  // Rotates weekly — same focus all week, changes every 7 days
  const weekIndex = Math.floor(Date.now() / (1000 * 60 * 60 * 24 * 7));
  const focus = focuses[weekIndex % focuses.length];

  const el = document.getElementById('weeklyFocusText'); // ← match your actual element ID
  if (el) el.textContent = focus;

  // Hide the loading state immediately
  const loader = document.getElementById('weeklyFocusLoader'); // ← match your loader ID
  if (loader) loader.style.display = 'none';
}

// Call on DOM ready — synchronous, instant
document.addEventListener('DOMContentLoaded', loadWeeklyFocus);
```

**Check your element IDs** — adjust `weeklyFocusText` and `weeklyFocusLoader` to match whatever IDs you actually have in the HTML for the focus display and its loading state.

**Time to ship:** 10 minutes.

## Fix 5 — Page load: kill the nudge toast on initial load

**Find your daily nudge show logic.** It likely looks like:

```js
// Somewhere in init or DOMContentLoaded:

// or
nudgeToast.style.display = 'block';
// or a timeout:

```

**Replace with — only show after the user has generated at least one prompt:**

```js
function maybeShowNudge() {
  // Only show if user has generated before (has history)
  const history = JSON.parse(localStorage.getItem('pmg_history') || '[]');
  if (history.length === 0) return; // ← new users never see it on load

  // For returning users, show max once per day
  const lastNudge = localStorage.getItem('pmg_nudge_date');
  const today = new Date().toISOString().slice(0, 10);
  if (lastNudge === today) return;

  localStorage.setItem('pmg_nudge_date', today);
  showNudge(); // your existing show function
}

document.addEventListener('DOMContentLoaded', maybeShowNudge);
```

**Adjust `pmg_history`** to match whatever localStorage key you use for prompt history. Search the codebase for `localStorage.setItem` near "history" to find the real key name.
