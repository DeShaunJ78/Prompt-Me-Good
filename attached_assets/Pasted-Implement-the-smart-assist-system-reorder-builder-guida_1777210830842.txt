========================
PART 1 — SMART ASSIST SYSTEM
========================

CORE GOAL:
Help users early without being intrusive.

STAGE 1 — MICRO NUDGE

Trigger:
- 3–5 seconds of inactivity
- User is in builder section
- No prompt generated
- User is not typing or interacting

Display:

Message:
“Need help getting started?”

Buttons:
- “Guide Me”
- “Try Demo”

Behavior:
- Guide Me → opens Guide Me flow
- Try Demo → triggers Use Demo Values + auto-generates prompt

Style:
- Small, subtle, non-blocking
- Bottom-center on mobile
- Bottom-right on desktop
- Smooth fade-in: 150–200ms

STAGE 2 — FULL HELPER

Trigger:
- 8–10 seconds total inactivity
OR
- Stage 1 shown but ignored

Display:

Title:
“Still stuck?”

Message:
“Use Guide Me for step-by-step setup, try a demo, explore Use Cases, or replay the tour.”

Buttons:
- “Guide Me”
- “Try Demo”
- “Use Cases”
- “Tour”
- “Dismiss”

Behavior:
- Guide Me → opens Guide Me flow
- Try Demo → loads demo + auto-generates prompt
- Use Cases → opens Use Cases modal
- Tour → replays guided tour
- Dismiss → hides helper for session

GLOBAL RULES:
- Show only once per session
- Do NOT show if user is typing, tour is active, demo is running, prompt already generated, or any modal is open
- Dismiss persists for session using sessionStorage

INTEGRATION SAFETY:
- Prevent duplicate timers
- Clear old inactivity timers before starting new ones
- Use one centralized timer manager
- Reset timers on scroll, click, typing, focus, or interaction
- Only one helper visible at a time
- Stage 2 cancels Stage 1
- Dismiss cancels both
- Do not conflict with guided tour, demo flow, toast notifications, or banners
- Use fixed positioning to avoid layout shifts
- Debounce activity listeners 100–150ms
- Use passive listeners where possible

MOBILE REQUIREMENTS:
- Do not overlap Generate Prompt, Use Demo Values, navigation, or keyboard
- Tap targets must be accessible
- No flickering
- No console errors
