CHANGE 6 — Session persistence: switch from localStorage to sessionStorage
---------------------------------------------------------------------------
In the session save and restore logic (wherever the prompt text, tuning selections, and generated result are saved), change every instance of:

  localStorage.getItem(...)
  localStorage.setItem(...)
  localStorage.removeItem(...)

to:

  sessionStorage.getItem(...)
  sessionStorage.setItem(...)
  sessionStorage.removeItem(...)

This is a one-word change everywhere. sessionStorage clears automatically when the tab is closed, giving users a clean slate on fresh visits while preserving the prompt when they switch apps temporarily (e.g., to paste into Gemini and return).

Do NOT change localStorage calls for user preferences, upgrade status, or saved vault prompts — only the temporary session state (current prompt, tuning picks, generated result).
