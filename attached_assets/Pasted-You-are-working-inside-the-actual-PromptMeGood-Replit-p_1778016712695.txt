You are working inside the actual PromptMeGood Replit project.

Goal:
Migrate PromptMeGood away from the current busy linear homepage/tool format into a ChatGPT/Claude-style conversational AI workspace.

The current site has strong features, but the current format creates mental overload because everything is stacked together: prompt builder, tuning, strength meter, photography tools, image generation, templates, vault, stats, Expert Mode, and marketing content.

The new direction is:
PromptMeGood should feel like an AI platform/workstation, not a long form page.

Core Product Model:
Users start with one clear input, PromptMeGood generates an extremely detailed powerful prompt, users can run it with AI, then they can refine, save, use Photography Suite, open Expert Mode, or generate a Master Actionable Plan.

Important:
Do not destroy existing work.
Do not rebuild blindly.
Do not remove working features.
Do not change database/storage behavior unless explicitly approved.
Do not introduce forced login.
Do not introduce new projects/sessions storage in this first migration.
Keep the current local-first Vault/history behavior.

Before building anything:
1. Inspect the current site structure.
2. Identify existing files/components for each current feature.
3. Map every current feature into the new ChatGPT-style chassis.
4. Create a visual preview/mockup of the new chassis.
5. Stop and wait for approval before implementing the full build.

Approved Chassis Concept:

Use a ChatGPT/Claude-style layout:

Left Sidebar:
- New Prompt / New Conversation
- Local Vault
- Recent prompts
- Saved prompts
- Favorites
- Templates
- Import / Export / Backup / Restore
- Theme or Appearance access if appropriate

Center Workspace:
- Conversation-style thread
- User messages
- Generated Prompt cards
- AI Result cards
- Master Actionable Plan cards
- Bottom composer/input bar

Right Contextual Tool Panel:
- Photography Suite
- Master Link controls
- Master Plan controls
- Optional tool-specific settings

Expert Command Center:
- Keep as drawer overlay
- Hidden by default
- Opened intentionally by user
- Do not place it permanently in the beginner flow

Landing Page And App Split:

Route `/`:
- Landing page
- Explains PromptMeGood
- Handles education, examples, trust, pricing/waitlist, and conversion
- Includes a powerful landing-page demo box
- Lets users type a rough idea
- Generates an extremely detailed high-quality prompt
- Lets users run/process the prompt with AI right there to see the result
- Then invites them into the full workstation

Route `/app`:
- Clean workstation
- No popups
- No forced onboarding
- No interruptive nudges
- No marketing clutter
- Just the tool

Landing Page Demo Requirements:
The landing page demo should not be weak or toy-like.
It should generate an extremely detailed, powerful prompt.
It should allow the user to run the generated prompt with AI right there.
After the AI result appears, show an inline CTA such as:

"Good start. Now unlock the full workstation."
"Open the full PromptMeGood Workstation to refine this prompt, shape the visual direction, generate a Master Actionable Plan, and save your work."

Do not use a popup for this.
Use an inline transition inside the result area.

Workstation Default Flow:

1. User opens `/app`.
2. User sees one clear composer/input area.
3. User types what they want.
4. PromptMeGood generates a powerful detailed prompt.
5. The generated prompt appears as a card in the conversation thread.
6. The card includes:
   - Run With AI
   - Copy
   - Save
   - Refine
   - Open Expert Mode if needed
7. If the user clicks Run With AI, the AI result appears in the same thread.
8. The user can improve, copy, save, or create a Master Actionable Plan.

Locked Architecture Decisions:

- Prompt Engine and Photography Suite are independent first.
- Each can generate individually.
- Master Link combines them only when the user wants.
- Main output/result area lives in the conversation workspace.
- Expert Mode lives as a drawer overlay.
- Storage remains local-first.
- No new projects/sessions in this first version.
- No forced login.
- No first-visit popups or nudges.
- Landing page explains; workstation executes.

Design Requirements:

- Keep current PromptMeGood brand coloring.
- Preserve deep teal / warm off-white direction.
- Preserve user ability to change site colors.
- No dashed borders.
- Use clean solid lines.
- Use premium dark-mode-first design.
- Use generous spacing.
- Avoid visual clutter.

- Use familiar AI-platform layout patterns.

No Popups / No Forced Onboarding:

Remove or avoid:
- First-visit popups.
- Automatic onboarding modals.
- Interruptive nudges.
- Forced tutorials.
- “Look here” overlays.

Keep:
