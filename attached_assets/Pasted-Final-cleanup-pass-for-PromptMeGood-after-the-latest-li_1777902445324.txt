Fix:
For any control that is visually hidden, collapsed, zero-size, inactive, offscreen, or in a non-current panel:

- add `inert`, OR
- set `disabled`, OR
- set `tabindex="-1"` and `aria-hidden="true"` until active.

Important:
Do not make visible/active controls inaccessible.
Only remove inactive hidden controls from keyboard/screen-reader access.
When a panel, modal, drawer, menu, or control becomes visible again, restore accessibility.

Acceptance scan:
Run this in desktop and mobile after fixes:

```js
[...document.querySelectorAll('a[href], button, input, textarea, select, details, [tabindex]:not([tabindex="-1"]), [role="button"], [contenteditable="true"]')]
  .filter(el => !el.disabled && el.getAttribute('aria-disabled') !== 'true' && el.getAttribute('tabindex') !== '-1')
  .filter(el => {
    const r = el.getBoundingClientRect();
    const cs = getComputedStyle(el);
    return (r.width === 0 || r.height === 0 || cs.display === 'none' || cs.visibility === 'hidden' || el.offsetParent === null)
      && !el.closest('[hidden]')
      && !el.closest('[inert]');
  })
