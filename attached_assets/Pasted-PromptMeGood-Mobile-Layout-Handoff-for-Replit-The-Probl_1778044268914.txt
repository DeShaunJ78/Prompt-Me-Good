PromptMeGood — Mobile Layout Handoff for Replit




The Problem

The current workstation chassis grid is hardcoded as fixed pixel widths:

CSS


.pmgv2-body {
  display: grid;
  grid-template-columns: 240px 669px 36px 320px; /* = 1,265px total */
}



On a 390px iPhone screen this does not collapse. The layout overflows and gets clipped by overflow-x on the body. The result is that the Photography Suite is completely invisible and unreachable on mobile. The composer textarea is buried deep in the page with no indication it exists. This is a critical usability failure on mobile.




The Solution — 4-Tier Responsive Chassis

Tier 1: Desktop (1280px and above) — No Change

Keep the existing 3-column layout exactly as it is. Only change the hardcoded pixel widths to use 1fr for the center column so it scales with the screen.

CSS


.pmgv2-body {
  display: grid;
  grid-template-columns: 240px 1fr 36px 320px;
  height: 100vh;
  overflow: hidden;
}






Tier 2: Tablet (768px–1279px) — Collapse the Left Rail

The left rail shrinks to an icon-only strip to give the center column and Photography Suite more room.

CSS


@media (max-width: 1279px) {
  .pmgv2-body {
    grid-template-columns: 56px 1fr 36px 280px;
  }
  /* Hide text labels in the left rail, keep icons only */
  .pmg-vault-labels,
  .pmg-template-labels,
  #history-list .history-label,
  #templates-grid .template-label {
    display: none;
  }
}






Tier 3: Mobile Landscape (600px–767px) — Stack + Drawer

The three columns stack vertically. The left rail becomes a bottom tab bar. The Photography Suite becomes a slide-up drawer triggered by a tab.

CSS


@media (max-width: 767px) {
  .pmgv2-body {
    display: flex;
    flex-direction: column;
    height: 100vh;
    width: 100vw;
    overflow: hidden;
  }

  /* Left rail becomes a bottom tab bar */
  .pmg-left-rail {
    order: 3;
    width: 100%;
    height: 56px;
    display: flex;
    flex-direction: row;
    justify-content: space-around;
    align-items: center;
    border-top: 1px solid var(--pmg-border);
    background: var(--pmg-panel-bg);
    position: fixed;
    bottom: 0;
    left: 0;
    right: 0;
    z-index: 300;
  }

  /* Photography Suite becomes a slide-up drawer */
  #pmg-photo-suite {
    position: fixed;
    bottom: 56px; /* sits above the tab bar */
    left: 0;
    right: 0;
    height: 65vh;
    background: var(--pmg-panel-bg);
    border-radius: 16px 16px 0 0;
    border-top: 1px solid var(--pmg-border);
    transform: translateY(100%);
    transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
    z-index: 200;
    overflow-y: auto;
  }

  /* When the Suite tab is active, slide it up */
  #pmg-photo-suite.is-open {
    transform: translateY(0);
  }

  /* Composer fixed above the tab bar */
  .composer-wrapper,
  #generateBtnTop,
  #goal {
    position: fixed;
    bottom: 56px;
    left: 0;
    right: 0;
    z-index: 100;
  }
}






Tier 4: Mobile Portrait (up to 599px) — Single Column, Full Focus

Full single-column layout. One tool at a time. Composer always visible. Photography Suite accessible via the "Suite" tab.

CSS


@media (max-width: 599px) {
  /* Kill horizontal overflow at the root */
  html, body {
    overflow-x: hidden;
    width: 100vw;
  }

  .pmgv2-body {
    display: flex;
    flex-direction: column;
    width: 100vw;
    overflow-x: hidden;
  }

  /* Center thread takes full width */
  #main {
    width: 100vw;
    padding-bottom: 160px; /* room for fixed composer + tab bar */
    overflow-y: auto;
  }

  /* Composer: always visible, fixed above tab bar */
  .composer-wrapper {
    position: fixed;
    bottom: 56px;
    left: 0;
    right: 0;
    z-index: 100;
    background: var(--pmg-panel-bg);
    padding: 12px 16px;
    border-top: 1px solid var(--pmg-border);
    box-shadow: 0 -8px 24px rgba(0, 0, 0, 0.4);
  }

  /* Minimum tap target size — Apple HIG + Google Material */
  button, [role="button"], select, label {
    min-height: 44px;
    min-width: 44px;
  }

  /* Prevent iOS auto-zoom on input focus */
  input, textarea, select {
    font-size: 16px !important;
  }

  /* Photography Suite: full-screen slide-up drawer */
  #pmg-photo-suite {
    position: fixed;
    bottom: 56px;
    left: 0;
    right: 0;
    height: 80vh;
    background: var(--pmg-panel-bg);
    border-radius: 16px 16px 0 0;
    border-top: 2px solid var(--pmg-accent);
    transform: translateY(100%);
    transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
    z-index: 200;
    overflow-y: auto;
    padding: 16px;
  }

  #pmg-photo-suite.is-open {
    transform: translateY(0);
  }
}






The Bottom Tab Bar — 3 Tabs

Add three tabs to the bottom navigation bar. These replace the left rail on mobile.

HTML


<!-- Add this inside .pmg-left-rail on mobile -->
<nav class="mobile-tab-bar">
  <button class="tab-btn active" data-tab="prompt">
    <span class="tab-icon">✍️</span>
    <span class="tab-label">Prompt</span>
  </button>
  <button class="tab-btn" data-tab="suite" id="open-suite-tab">
    <span class="tab-icon">📸</span>
    <span class="tab-label">Suite</span>
  </button>
  <button class="tab-btn" data-tab="vault">
    <span class="tab-icon">🗄️</span>
    <span class="tab-label">Vault</span>
  </button>
</nav>



CSS


.mobile-tab-bar {
  display: flex;
  width: 100%;
  justify-content: space-around;
}

.tab-btn {
  display: flex;
  flex-direction: column;
  align-items: center;
  gap: 2px;
  padding: 8px 16px;
  background: none;
  border: none;
  color: var(--pmg-text-muted);
  font-size: 11px;
  cursor: pointer;
}

.tab-btn.active {
  color: var(--pmg-accent);
}

.tab-icon {
  font-size: 20px;
}



JavaScript


// Tab switching logic
document.querySelectorAll('.tab-btn').forEach(btn => {
  btn.addEventListener('click', () => {
    const tab = btn.dataset.tab;

    // Remove active from all tabs
    document.querySelectorAll('.tab-btn').forEach(b => b.classList.remove('active'));
    btn.classList.add('active');

    if (tab === 'suite') {
      // Slide up the Photography Suite drawer
      document.getElementById('pmg-photo-suite').classList.add('is-open');
    } else {
      // Close the drawer
      document.getElementById('pmg-photo-suite').classList.remove('is-open');
    }

    if (tab === 'vault') {
      // Show the vault panel
      document.querySelector('.pmg-left-rail .vault-content').style.display = 'block';
    }
  });
});






Acceptance Criteria

Replit must verify all of the following before marking this complete:

1.
On a 390px wide screen (iPhone 14 Pro), the workstation fits within the viewport with zero horizontal scrolling.

2.
The composer textarea and "Fix My Prompt" button are visible without scrolling on all mobile screen sizes.

3.
The Photography Suite is reachable on mobile via the "Suite" tab — it must not be cut off or hidden.

4.
All tap targets are a minimum of 44px × 44px.

5.
Input and textarea font sizes are a minimum of 16px to prevent iOS auto-zoom.

6.
The slide-up drawer animation for the Photography Suite is smooth (no jank or flicker).

7.
The bottom tab bar does not overlap the composer or any content.
