# Self-contained UX scripts

Each lives at `artifacts/promptmegood/public/scripts/` with `?nameKey` URL kill-switch + `localStorage.pmg_{name}_disable='1'` per-device kill-switch.

When adding a new post-paint mounter, **subscribe to `pmgMountBus`** instead of adding another body-wide `MutationObserver` — see `pmg-mount-bus.js` below.

---

*   **`pmg-storyboard.js` (sb-3)** — Storyboard Studio modal: `/api/storyboard` returns 5 cinematic prompts, then 5 parallel `/api/image` calls (capped at 2 in flight to avoid free-tier 429s). Session token (`_sessionId`) is bumped on every open/close so stale in-flight responses don't mutate the closed modal or burn more `/api/image` credits. **sb-2 added:** (1) per-frame **🔄 Regenerate** button — silent one-tap, re-runs `generateOne(i, _sessionId)` against the LIVE session token (not bumped) so the result lands in the open modal; disabled while pending. (2) **🖨 Export PDF** — opens a new window with a self-contained 2-up portrait print document (concept verbatim + localized date + footer), waits for `<img>` to load (or 6s timeout), then auto-`window.print()`. Zero deps. Pop-up blocked → button briefly shows "⚠️ Pop-up blocked". `@page letter portrait; margin: 0.6in 0.6in 0.8in` and `break-inside: avoid` per panel. **sb-3 adds:** (1) per-frame **✏️ Edit** button — inline DOM swap (no modal): replaces `.pmg-sb-panel-text` with a textarea prefilled from `_currentPanels[i].prompt` + "Regenerate with this prompt" / "Cancel" buttons. Submit mutates `p.prompt` (canonical — Send-to-Video / Copy / PDF / Vault all see the edited text) then calls `regenerateOne(i)`. Empty submit → `pmg-sb-shake` animation, no mutation. Esc inside the textarea → cancel (does NOT close modal — `e.stopPropagation()` on the keydown handler). Cmd/Ctrl+Enter inside the textarea → submit. (2) **💾 Save to Vault** button in actions row — paired write to `localStorage['promptmegood:history:v1']`: parent `kind:'built'` (concept as `prompt`, `storyboardParent:true` marker) + child `kind:'storyboard'` (`parentId` link, `concept`, full `panels` array with `{prompt, imageUrl|null, status}`). Pending/failed frames save with `imageUrl:null` (silent, honest about state). 10-second dedupe guard: scans for prior storyboard child with same `concept` saved within `DEDUPE_MS=10000` and bails with "✓ Already in Vault" — catches double-taps without blocking intentional re-saves. Fires `pmg:vault-saved` event so Stats / draft-recovery banners refresh. Existing Vault renderer treats unknown `kind:'storyboard'` as `'built'` (per `pmg-save-result.js` L22 comment) so entry shows as a normal text entry until a future Vault rework adds storyboard preview. Helpers `panelImgInner(p,i)` + `panelTextInner(p,i)` + new `panelEditorInner(p,i)` share a single source of truth for panel markup. Disable: `?nostoryboard`-style kill-switches not yet wired (it's the only path to the feature). API: `window.openStoryboard(concept)`.

*   **`pmg-chip-tooltips.js` (cti-1)** — Plain-English outcome tooltips on every `#pmg-photo-suite .pmg-photo-pill`. 104-entry `TIPS` map keyed by exact pill label (Style 12 / Camera & Lens 33 / Lighting 17 / Composition 17 / Camera Angle 10 / Palette 11 / Aspect 4). On apply: sets `data-pmg-tip` (visual layer), `title` (iOS long-press + a11y baseline), and `aria-label="<label>. <tip>"` (screen readers). Desktop reveal via `@media (hover: hover) :hover/:focus-visible` on `[data-pmg-tip]::after`. Touch reveal via `.is-tip-open` toggled by a delegated `click` listener on `document` (bound once, only when `matchMedia('(hover: none)').matches`); tap pins tooltip + 4s auto-dismiss timer, tap outside / Esc / new chip dismisses early. Critical: never `preventDefault` — pmg-ux's existing pill toggle (`is-active` class swap) keeps working alongside. The pill's existing `::before { content: "✓" }` (set by pmg-ux on `.is-active`) is preserved by reusing `::before` for the tooltip arrow ONLY when not active. Edge-aware alignment via `pmg-tip-align-left/right` classes set just-in-time from `getBoundingClientRect()` so tooltips on the leftmost/rightmost pills don't overflow the viewport. Mounting: subscribes to `pmgMountBus` when active + 500ms scan loop for 60s + scoped `MutationObserver` on `#pmg-photo-suite` once it exists (Photo Suite re-renders on Surprise Me / preset apply / saved-combo apply). Idempotent via `data-pmg-tip-bound="1"` flag. Disable: `?notooltips`, `localStorage.pmg_chip_tooltips_disable='1'`. API: `window.pmgChipTooltips.{apply, refresh, dismiss, getTip}`.

*   **`pmg-dna-card.js` (dna-v2-1)** — v2 of the 1080×1350 Prompt DNA Card. Loaded BEFORE `pmg-visual-studio.js` so visual-studio's `composeDnaCard()` can delegate to `window.pmgDnaCardV2.composeDnaCardV2(imgSrc, promptText)` when present. Layout: image 0–900 / score band 900–1010 / techniques 1010–1090 / prompt 1090–1310 / watermark 1310–1350. Score is a heuristic 0–100 (length sweet spot, photography-vocab hits, anchor prepositions, comma-stack bonus, vague-word penalty) → grade A/B/C/D/F + accent color. Techniques: insertion-ordered keyword buckets (cinematic / golden hour / shallow DOF / hard light / soft light / studio / film grain / color grade / macro / wide / telephoto / documentary / long exposure / high contrast / symmetry / negative space / rule of thirds), top 5 returned, "+N more" tail when chip row would overflow. Watermark: "🧬 Built with PromptMeGood · promptmegood.com" on one line, falls back to two lines when card is narrowed by future layout changes. Score + technique helpers exposed as `window.pmgDnaCardV2.{scorePrompt, detectTechniques}` so a future backend `/api/score-prompt` route can swap the implementation without touching layout code. Image CORS / decode failure renders the fallback gradient panel instead of erroring. Disable: `?nodnacard`, `localStorage.pmg_dna_card_disable='1'` — visual-studio falls through to the legacy v1 composer it still ships inline.

*   **`pmg-mount-bus.js` (mb-1)** — Shared "DOM-ready-for-mounters" coordinator. Replaces N independent `MutationObserver(document.body, {childList:true, subtree:true})` boot observers with ONE shared observer + rAF coalescing + 30s auto-disconnect. Mounters call `if (window.pmgMountBus && window.pmgMountBus.isActive()) { window.pmgMountBus.subscribe(tick); }` and fall back to their own observer otherwise. Already wired into: `pmg-coach`, `pmg-quick-chips`, `pmg-target-platform`, `pmg-text-feedback`, `pmg-voice`. **When adding a new post-paint mounter, USE THE BUS** — don't add another body-wide observer or you reintroduce the mobile boot stutter. NOT wired into `pmg-guided-intake` (its observer is conditional on `_mo` state — left intentionally untouched). Loaded BEFORE consumers via `<script src="/scripts/pmg-mount-bus.js?v=mb-1" defer>` at app.html L12510. API: `window.pmgMountBus.{subscribe, isActive}`.

*   **`pmg-voice.js` (vox-1)** — Web Speech mic + 9-lang picker mounted under `#goal` / `#pmg-vs-image-goal` / `#pmg-vs-video-goal` via 30s MutationObserver. No-ops when no `webkitSpeechRecognition`. Lang → `localStorage['pmg.voice.lang.v1']`. API: `window.pmgVoice.{setLang, getLang, supported}`.

*   **`pmg-command-palette.js` (cmd-1)** — Global ⌘K/Ctrl+K overlay (`#pmg-cmd-palette`). Fuzzy search across Navigate / Actions / Tours / Help / Vault (last 20 from `promptmegood:history:v1`). Calls only existing public APIs. ALSO owns the global `?`-key handler that opens the cheatsheet. API: `window.pmgCommandPalette.{open, close, toggle, isOpen}`.

*   **`pmg-cheatsheet.js` (cs-1)** — Shortcuts modal (`#pmg-cheatsheet`). Mod key auto-detects platform. Opened by the `?` key handler in command-palette or `window.pmgCheatsheet.open()`. Zero global key listeners of its own (besides Esc-when-open).

*   **`pmg-stats.js` (stats-1)** — Calm dismissible Vault stats card above `#history-list`: total saved, this-month, day-streak, top 3 tags. Reads only `promptmegood:history:v1`. Streak allows today OR yesterday so it doesn't vanish at 12:01am. Hides when `total===0`. Refreshes on `pmg:vault-saved`. Dismissed-stays-dismissed via `localStorage['pmg.stats.dismissed.v1']`. API: `window.pmgStats.{render, dismiss, reset}`.

*   **`pmg-text-feedback.js` (txt-fb-2-clarity)** — Live feedback card under `#goal`, **desktop-only** (text panel + ≥769px — hidden on Photo/Video via `body[data-active-panel]` and on mobile by the pre-existing `pmg-chassis-v3.css` L1474-1479 rule, intentional cv3-17 friction fix). Five pieces: (1) tuning summary chips (non-default category/tone/format/length/personality), (2) quality meter (Too vague / Getting there / Specific enough — based on length + vague-word count), (3) **Clarity Checklist popover (txt-fb-2)** — ⓘ button next to the meter toggles a 4-item tooltip (Persona / Goal / Context / Constraints) with ✓/○ marks driven by `CLARITY_ITEMS[].test(text)` regex; click-outside + Esc close. Inline styles via `CLARITY_STYLE_ID` so we don't have to bump the external `.css` cache-buster. (4) char + ~token counter, (5) vague-word linter chips. Re-binds on a 30s MutationObserver. Disable: `?nofeedback`, `localStorage.pmg_feedback_disable='1'`. API: `window.pmgTextFeedback.{refresh, getState, dismiss}`.

*   **`pmg-coach.js` (coach-1)** — Photo + Video Coach meter mounted under each visual panel's goal textarea (after `#pmg-vs-image-live-preview` / `#pmg-vs-video-live-preview` if present, else after the goal). Shows a quality meter + ⓘ Clarity Checklist popover. Photo checklist: Subject / Lighting / Lens or Camera / Background. Video checklist: Subject / Action / Camera Direction / Visual Style. Detection via per-item `test(text)` regex; ✓ when met, ○ when not. Inline styles, theme tokens (no hardcoded teal). Mounts via 30s MutationObserver since visual-studio builds the panels async. Disable: `?nocoach`, `localStorage.pmg_coach_disable='1'`. API: `window.pmgCoach.{refresh, getState, dismiss}`.

*   **`pmg-quick-chips.js` (chips-1)** — Photo + Video quick-add chip row mounted **after** `pmg-coach` (or live preview / goal as fallback). Photo: `Leica Look`. Video: `Viral`, `Luxury`. Tapping a chip APPENDS a single descriptor sentence to the goal textarea (with ` — ` separator unless the existing text ends with `.!?`), fires `input` + `change` events so the existing visual-studio Live Preview observer picks it up. Briefly highlights the chip via `[data-just-added="1"]`. Inline styles, theme tokens. Disable: `?nochips`, `localStorage.pmg_chips_disable='1'`. API: `window.pmgQuickChips.{refresh, dismiss}`.

*   **`pmg-target-platform.js` (tp-1)** — Text-only Target Platform `<select>` injected after the `#outputFormat` field's `.field` wrapper inside the 🎛️ Tune Your Prompt section. Options: empty / LinkedIn / Email / Blog / Ad Copy. **No backend schema change** — instead, on change `syncDetails()` strips any prior `[PromptMeGood: optimize this prompt for X — …]` sentinel from `#details.value` (regex `MARKER_RE = /\s*\[PromptMeGood: optimize this prompt for [^\]]+\]\s*/g`) and appends the new one (or just strips if "No platform" picked). Hint is fully visible in the Extra Details textarea — user can edit/delete. Selection persists to `localStorage['pmg.target.platform.v1']`. Re-applied on boot. Disable: `?notarget`, `localStorage.pmg_target_platform_disable='1'`. API: `window.pmgTargetPlatform.{get, set, refresh, dismiss}`.

*   **`pmg-magic-flow.js` (mf-1)** — One-Click Magic Flow that bridges the two-step Build → Generate into a single click on `#analyze-btn` ("✨ Build My Prompt"). **Capture-phase listener** (third arg `true`) so it snapshots the original button label BEFORE the chassis bubble handler sets `analyzeBtn.style.display='none'`. Adds `body.pmg-magic-active` which (a) keeps the Build button visible despite the chassis inline `display:none` (CSS rule uses `!important` — chassis inline does not), (b) hides `#pmgv3-reanalyze` (Re-analyze link) and `#pmgv3-ready-label` (Ready badge) so they don't compete with the bridge UI. Swaps label → "✨ Tuning your idea…" → after `AUTO_TUNE_CAP_MS=3500` fires `#generateBtn.click()` and swaps to "✨ Generating your prompt…". The 3.5s cap intentionally does NOT await `/api/auto-tune` — chassis writes picks asynchronously and the form submit reads whatever the selects say at submit time. Watches `body.pmg-has-result` (canonical "result landed" signal app.html sets) via class-attribute MutationObserver to end the flow; 30s hard ceiling as fallback. Top progress bar `#pmg-magic-progress` is fixed-position with `data-pmg-overlay-root` (chassis universal-hide rule erases unmarked body-level elements), uses `var(--color-primary, #3ee0a0)` + `color-mix` (no hardcoded teal). Subscribes to `pmgMountBus` when active, falls back to scoped 30s observer. Idea < 4 chars no-ops (lets normal validation run). Re-clicking analyze-btn while active no-ops via `state.active` guard. Disable: `?nomagic`, `localStorage.pmg_magic_disable='1'`. API: `window.pmgMagicFlow.{end, isActive}`.
