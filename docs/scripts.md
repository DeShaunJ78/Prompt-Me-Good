Each lives at `artifacts/promptmegood/public/scripts/` with `?nameKey` URL kill-switch + `localStorage.pmg_{name}_disable='1'` per-device kill-switch.

When adding a new post-paint mounter, **subscribe to `pmgMountBus`** instead of adding another body-wide `MutationObserver` вАФ see `pmg-mount-bus.js` below.

*   **`pmg-storyboard.js` (sb-2)** вАФ Storyboard Studio modal: `/api/storyboard` returns 5 cinematic prompts, then 5 parallel `/api/image` calls (capped at 2 in flight to avoid free-tier 429s). Session token (`_sessionId`) is bumped on every open/close so stale in-flight responses don't mutate the closed modal or burn more `/api/image` credits. **sb-2 additions:** (1) per-frame **рЯФД Regenerate** button under the prompt text вАФ silent, one-tap, sets that panel's status to `pending` and re-runs `generateOne(i, _sessionId)` against the LIVE session token (not bumped) so the result lands in the open modal; disabled while pending. (2) **рЯЦ® Export PDF** button in actions row вАФ opens a new window containing a self-contained 2-up portrait print document (title block uses concept text verbatim + localized date + "рЯОЮпЄП Storyboard" header + "Built with PromptMeGood ¬Ј promptmegood.com" footer), waits for all `<img>` to load (or 6s hard timeout), then auto-calls `window.print()`. Zero deps вАФ relies on the browser's "Save as PDF" print destination. Pop-up blocked вЖТ button briefly shows "вЪ†пЄП Pop-up blocked вАФ allow & retry". Print CSS uses `@page letter portrait; margin: 0.6in 0.6in 0.8in` and `break-inside: avoid` per panel so frames never split across pages. Helpers `panelImgInner(p,i)` + `panelTextInner(p,i)` are reused by both initial render and `updatePanel(i)` so regen state stays in sync. Disable: `?nostoryboard`-style kill-switches not yet wired (it's the only path to the feature). API: `window.openStoryboard(concept)`.

*   **`pmg-chip-tooltips.js` (cti-1)** вАФ Plain-English outcome tooltips on every `#pmg-photo-suite .pmg-photo-pill`. 104-entry `TIPS` map keyed by exact pill label (Style 12 / Camera & Lens 33 / Lighting 17 / Composition 17 / Camera Angle 10 / Palette 11 / Aspect 4). On apply: sets `data-pmg-tip` (visual layer), `title` (iOS long-press + a11y baseline), and `aria-label="<label>. <tip>"` (screen readers). Desktop reveal via `@media (hover: hover) :hover/:focus-visible` on `[data-pmg-tip]::after`. Touch reveal via `.is-tip-open` toggled by a delegated `click` listener on `document` (bound once, only when `matchMedia('(hover: none)').matches`); tap pins tooltip + 4s auto-dismiss timer, tap outside / Esc / new chip dismisses early. Critical: never `preventDefault` вАФ pmg-ux's existing pill toggle (`is-active` class swap) keeps working alongside. The pill's existing `::before { content: "вЬУ" }` (set by pmg-ux on `.is-active`) is preserved by reusing `::before` for the tooltip arrow ONLY when not active. Edge-aware alignment via `pmg-tip-align-left/right` classes set just-in-time from `getBoundingClientRect()` so tooltips on the leftmost/rightmost pills don't overflow the viewport. Mounting: subscribes to `pmgMountBus` when active + 500ms scan loop for 60s + scoped `MutationObserver` on `#pmg-photo-suite` once it exists (Photo Suite re-renders on Surprise Me / preset apply / saved-combo apply). Idempotent via `data-pmg-tip-bound="1"` flag. Disable: `?notooltips`, `localStorage.pmg_chip_tooltips_disable='1'`. API: `window.pmgChipTooltips.{apply, refresh, dismiss, getTip}`.

*   **`pmg-mount-bus.js` (mb-1)** вАФ Shared "DOM-ready-for-mounters" coordinator. Replaces N independent `MutationObserver(document.body, {childList:true, subtree:true})` boot observers with ONE shared observer + rAF coalescing + 30s auto-disconnect. Mounters call `if (window.pmgMountBus && window.pmgMountBus.isActive()) { window.pmgMountBus.subscribe(tick); }` and fall back to their own observer otherwise. Already wired into: `pmg-coach`, `pmg-quick-chips`, `pmg-target-platform`, `pmg-text-feedback`, `pmg-voice`. **When adding a new post-paint mounter, USE THE BUS** вАФ don't add another body-wide observer or you reintroduce the mobile boot stutter. NOT wired into `pmg-guided-intake` (its observer is conditional on `_mo` state вАФ left intentionally untouched). Loaded BEFORE consumers via `<script src="/scripts/pmg-mount-bus.js?v=mb-1" defer>` at app.html L12510. API: `window.pmgMountBus.{subscribe, isActive}`.
