# Self-contained UX scripts

Each lives at `artifacts/promptmegood/public/scripts/` with `?nameKey` URL kill-switch + `localStorage.pmg_{name}_disable='1'` per-device kill-switch.

When adding a new post-paint mounter, **subscribe to `pmgMountBus`** instead of adding another body-wide `MutationObserver` ‚Äî see `pmg-mount-bus.js` below.

*   **`pmg-visual-studio.js` (re-3 reverse)** ‚Äî Reverse Engineer sub-tab UX flipped from silent auto-inject to **explicit accept/discard**. After `/api/vision-analyze` resolves, the decoded prompt + suite-setting count surface in a `#pmg-vs-reverse-preview` card with a primary `Use this prompt ‚Üí` button (#pmg-vs-reverse-use-btn) and a secondary `Discard` link (#pmg-vs-reverse-discard-btn). Goal textarea (#pmg-vs-image-goal) is **not touched** until the user clicks Use; Discard clears the preview without injecting. **Race-guard invariant:** monotonic `_reverseReqSeq` token ‚Äî incremented on every upload AND on Discard ‚Äî gates DOM/state mutations so a slow first analyze can't overwrite a fresh second one or resurrect a discarded preview. Stale responses are silently dropped (no toast). On Use: injects ‚Üí applies suite settings ‚Üí `setPhotoSubMode('create')` ‚Üí status reads "‚úì Prompt loaded into Create New ‚Äî tweak, build, then Generate." Outcome-focused intro copy: bold "Steal any photo's look in one click." + "Drop a photo ‚Üí get a prompt you can edit, riff on, and re-generate ‚Äî composition, lighting, palette, and lens all decoded for you." Cache buster: `?v=re-3` on both CSS + JS in `app.html`.

*   **`pmg-chip-tooltips.js` (cti-1)** ‚Äî Plain-English outcome tooltips on every `#pmg-photo-suite .pmg-photo-pill`. 104-entry `TIPS` map keyed by exact pill label (Style 12 / Camera & Lens 33 / Lighting 17 / Composition 17 / Camera Angle 10 / Palette 11 / Aspect 4). On apply: sets `data-pmg-tip` (visual layer), `title` (iOS long-press + a11y baseline), and `aria-label="<label>. <tip>"` (screen readers). Edge-aware alignment via `pmg-tip-align-left/right` classes set just-in-time from `getBoundingClientRect()` so tooltips on the leftmost/rightmost pills don't overflow the viewport. Mounting: subscribes to `pmgMountBus` when active + 500ms scan loop for 60s + scoped `MutationObserver` on `#pmg-photo-suite` once it exists (Photo Suite re-renders on Surprise Me / preset apply / saved-combo apply). *   **`pmg-mount-bus.js` (mb-1)** ‚Äî Shared "DOM-ready-for-mounters" coordinator. Replaces N independent `MutationObserver(document.body, {childList:true, subtree:true})` boot observers with ONE shared observer + rAF coalescing + 30s auto-disconnect. Mounters call `if (window.pmgMountBus && window.pmgMountBus.isActive()) { window.pmgMountBus.subscribe(tick); }` and fall back to their own observer otherwise. Already wired into: `pmg-coach`, `pmg-quick-chips`, `pmg-target-platform`, `pmg-text-feedback`, `pmg-voice`. **When adding a new post-paint mounter, USE THE BUS** ‚Äî don't add another body-wide observer or you reintroduce the mobile boot stutter. NOT wired into `pmg-guided-intake` (its observer is conditional on `_mo` state ‚Äî left intentionally untouched). Loaded BEFORE consumers via `<script src="/scripts/pmg-mount-bus.js?v=mb-1" defer>` at app.html L12510. API: `window.pmgMountBus.{subscribe, isActive}`.
